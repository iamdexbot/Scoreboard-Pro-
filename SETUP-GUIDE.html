Complete Setup Guide ‚Äî Supabase + Auth

1 Create a Free Supabase Project

Go to supabase.com and sign in with GitHub or Google. Click New Project.

Choose any name (e.g. "hoops-pro"), pick a region close to you, and set a database password. Click Create new project and wait ~1 minute.

2 Get Your API Keys

In your Supabase project, go to Settings ‚Üí API. You need two values:

ValueWhere to find it
Project URLLooks like: https://abcdefgh.supabase.co
anon / public keyLong string starting with "eyJ..."

3 Edit supabase-config.js

Open supabase-config.js in any text editor and replace the two placeholder values:

const SUPABASE_URL = 'https://YOUR_PROJECT_ID.supabase.co'; const SUPABASE_ANON = 'YOUR_ANON_PUBLIC_KEY';

Replace with your actual values:

const SUPABASE_URL = 'https://abcdefgh.supabase.co'; const SUPABASE_ANON = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
Note: The anon key is safe to include in frontend code ‚Äî it's read-only by default and protected by Row Level Security.

4 Run the Database Setup SQL

In your Supabase project, go to SQL Editor ‚Üí New Query. Open the file supabase-setup.sql, copy its entire contents, paste it into the editor, and click Run.

‚úì This creates: profiles, game_state, pro_data, upcoming_games tables ‚Äî all with Row Level Security so each user only sees their own data.

5 Enable Google Sign-In (optional)

In Supabase ‚Üí Authentication ‚Üí Providers ‚Üí Google. Toggle it on.

You'll need a Google OAuth Client ID. Go to console.cloud.google.com ‚Üí APIs & Services ‚Üí Credentials ‚Üí Create OAuth 2.0 Client ID. Set the redirect URI to:

https://YOUR_PROJECT_ID.supabase.co/auth/v1/callback

Paste the Client ID and Secret back into Supabase's Google provider settings.

Skip this if you only want email/password login. Email login works out of the box with no extra config.

6 Host Your Files

All files must be served from the same origin (not opened as local files) for auth to work. Recommended free options:

OptionHow
GitHub PagesPush files to a GitHub repo ‚Üí Settings ‚Üí Pages ‚Üí Deploy from main branch
NetlifyDrag the folder to netlify.com/drop ‚Äî done in 10 seconds
VercelConnect your GitHub repo at vercel.com

After deploying, add your site URL to Supabase ‚Üí Authentication ‚Üí URL Configuration ‚Üí Site URL.

7 Enable Email Confirmations (optional)

By default Supabase requires users to confirm their email. To skip this for testing: Supabase ‚Üí Authentication ‚Üí Email ‚Üí toggle off "Enable email confirmations".
